The near-miss cases all reveal themselves as one-off / throwaway / mechanical, which that clause catches.
- **Honest caveats (Evidence Contract → Known Blind Spots)**:
  - Single model (`deepseek-v4-pro`). Trigger accuracy may differ on other models; this validates *this* model's read of the description, not a model-independent property. A second model run would strengthen the claim.
  - 20 queries is a small set. The near-miss bucket (the real discriminator test) is 7 total — passing all 7 is meaningful but not exhaustive.
  - The description-only protocol is a *unit test for the description string*. The host's native trigger runtime (how opencode/pi actually decides to load an installed skill) is a separate integration question not tested here, as eval-guide notes.
  - One transient empty-response during the first run (case 6) — handled by the runner's retry loop; the final log for every id contains exactly one `VERDICT:` line (verified).
